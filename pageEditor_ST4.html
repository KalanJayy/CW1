User Profile

Technical Description

The User Profile Page was implemented using HTML and CSS for styling, with div elements used to create a structured layout and flexbox for responsive design. A skills progress bar was created using div elements with animated width transitions to showcase progress. The volunteering section was positioned at the top-right corner using position: absolute. A table was embedded using the table tag, while the contact form includes fields for name, email, and message with built-in validation. Accessibility was considered by including alt attributes for images. Additionally, the page passed W3C HTML and CSS validation, with only minor warnings.

Accessibility

Alt Attributes: Provided for all images to support screen readers.
Semantic HTML: Good use of header, footer, and main tags to enhance structure.

Sitemap Page

Technical Description

The Sitemap Page was implemented using SVG graphics, with rect and text elements to create a visual sitemap. CSS hover effects were applied to change the color of elements when hovered, enhancing the user experience. For accessibility, aria-labels were added to interactive elements, and the color contrast was ensured to meet WCAG guidelines. The page passed W3C validation with no errors, ensuring compliance with web standards.

Content Page

Technical Description

The Content Page was structured using semantic HTML elements like section and article for better readability and organization. Internal navigation was implemented with anchor links to enable smooth scrolling throughout the page. A fixed-position "Go-to-Top" button was added for quick navigation. For responsive design, CSS Grid Layout was used to adjust the page layout on different screen sizes. Accessibility features include ensuring a proper heading order from h1 to h6 and providing descriptive link texts instead of generic phrases like "Click here." The page passed validation with minor warnings due to missing alt attributes on some images.

Challenges and Lessons Learned

Several challenges were encountered during the development process. One challenge was aligning the volunteering section on the User Profile page, which was resolved by using position: absolute; right: 20px; top: 20px; to precisely position it. Another challenge was making the SVG sitemap interactive, which was addressed by using JavaScript to add onclick events to the nodes, allowing for dynamic user interaction. Lastly, ensuring accessibility compliance was a challenge, but this was solved by adding ARIA attributes and improving the color contrast to meet accessibility standards

Compliance

This project provided valuable insights into key aspects of web development. I learned the importance of using structured HTML and CSS for creating responsive designs, as well as how to integrate SVG graphics for interactive sitemaps. I also gained a deeper understanding of the role ARIA attributes play in improving accessibility and ensuring that websites are usable by everyone. Additionally, the project highlighted how validation tools help maintain web standards, ensuring that the site is both functional and complian with industry best practices.
